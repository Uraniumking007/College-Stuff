Practical 9 - Basic Node.js Programs

Demonstrating basic Node.js programs and how to execute them

Introduction to Node.js

Node.js is a JavaScript runtime built on Chrome's V8 engine. It allows you to run JavaScript on the server side.

To run Node.js programs:
  1. Install Node.js from nodejs.org
  2. Save code in a .js file (e.g., app.js)
  3. Open terminal and run: node filename.js
Node.js Backend Examples

Comprehensive backend examples demonstrating Node.js server-side programming

📁 Location: All backend examples are in backend/practical9/ directory
1. Basic HTTP Server

01-basic-http-server.cjs

Simple HTTP server using Node's built-in http module with route handling

  • Route handling (/home, /about, /contact)
  • Custom 404 pages
  • Runs on port 3001
2. File Server

02-file-server.cjs

Static file server with automatic MIME type detection

  • Serves HTML, CSS, JS files
  • MIME type handling
  • Runs on port 3002
3. REST API Server

03-rest-api-server.cjs

Full RESTful API with CRUD operations

  • GET, POST, PUT, DELETE endpoints
  • In-memory data store
  • Runs on port 3003
4. Streams Example

04-streams-example.cjs

Memory-efficient file operations using streams

  • Read/write streams
  • Pipe operations
  • Runs on port 3004
5. Event Emitter

05-event-emitter.cjs

Event-driven programming with custom events and listeners

  • Custom events (login, logout, error)
  • Event listeners and handlers
  • Runs on port 3005
📋 How to Run Backend Examples
📁 Location: backend/practical9/

Step-by-Step Instructions:

  1. Open terminal/command prompt
  2. Navigate to the practical9 directory: 
    cd backend/practical9
  3. Run any example: 
    node 01-basic-http-server.cjs
  4. Open browser and visit the indicated port
  5. Press Ctrl+C to stop the server
🚀 Quick Start Commands:

Run:

cd backend/practical9
node 01-basic-http-server.cjs

Test in browser:

http://localhost:3001/
http://localhost:3001/about
http://localhost:3001/api

Test with curl:

curl http://localhost:3001/
curl http://localhost:3001/api

Run:

cd backend/practical9
node 02-file-server.cjs

Test:

curl http://localhost:3002/index.html

Run:

cd backend/practical9
node 03-rest-api-server.cjs

Test endpoints:

# Get all users
curl http://localhost:3003/api/users

# Get single user
curl http://localhost:3003/api/users/1

# Create user
curl -X POST http://localhost:3003/api/users \\
  -H "Content-Type: application/json" \\
  -d '{"name":"Alice","email":"alice@example.com"}'

# Update user
curl -X PUT http://localhost:3003/api/users/1 \\
  -H "Content-Type: application/json" \\
  -d '{"name":"John Updated"}'

# Delete user
curl -X DELETE http://localhost:3003/api/users/1

Run:

cd backend/practical9
node 04-streams-example.cjs

Test:

curl http://localhost:3004/read-stream
curl http://localhost:3004/write-stream
curl http://localhost:3004/pipe-stream

Run:

cd backend/practical9
node 05-event-emitter.cjs

Test (watch server console for events!):

curl http://localhost:3005/login
curl http://localhost:3005/logout
curl http://localhost:3005/error
💡 Tips:
  • Multiple servers: You can run multiple examples simultaneously (different ports: 3001-3005)
  • Stop server: Press Ctrl+C in the terminal
  • Watch logs: Check terminal for request logs and event emissions
  • Testing: Use browser, curl, or Postman to test endpoints
⚠️ Common Issues:

Error: Error: listen EADDRINUSE: address already in use :::PORT

Solution:

  • Stop the other process using that port
  • Or change the PORT variable in the code
  • Find process: lsof -i :3001 (Mac/Linux) or netstat -ano | findstr :3001 (Windows)

Error: ENOENT: no such file or directory

Solution: Make sure you're in the correct directory (backend/practical9)
